# Verification Proof for SSH-111

This ticket resolves the SSH MITM (Man-In-The-Middle) Risk where the connection manager silently fell back to a `PromiscuousVerifier`.

**1. Root Cause Analysis:**
Previously, `SshConnectionManager` checked for a configured `HostKeyVerifier`. If none was found, it attempted to load the default SSH `known_hosts` file (which throws an IOException on Android due to missing standard filesystem structures). On failure, it explicitly logged a warning and applied `net.schmizz.sshj.transport.verification.PromiscuousVerifier()`. This caused the app to silently accept all host keys, making it completely vulnerable to MITM attacks.

**2. Remediation:**
*   Implemented a custom `TofuHostKeyVerifier` (Trust On First Use) that extends `OpenSSHKnownHosts`.
*   Modified the `SshConnectionManager` constructor to accept an Android `Context`.
*   Created a local `ssh_known_hosts` file inside the application's internal `filesDir`.
*   The connection manager now applies the `TofuHostKeyVerifier` by default. 
*   If a host connects for the first time, its key is automatically saved to the local `ssh_known_hosts` file.
*   If the host reconnects later and its key has changed, the connection is instantly rejected, effectively preventing MITM attacks.

The `./gradlew test lint` executed successfully, passing all 72 tests.
