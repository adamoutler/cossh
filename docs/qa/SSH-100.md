# SSH-100: Misc UX and Auth Fallback Improvements

## User Story
*As a user, I expect continuous volume key font scaling without needing to tap first, a safety confirmation when deleting connections, and an auth flow that tries both key and password seamlessly.*

## Implementation Details

1. **Continuous Volume Key Scaling:**
   - Modified `TerminalScreenContent` to use `rememberUpdatedState` for `currentFontSize` and `onUpdateFontSize`. The `TerminalViewClient` previously captured initial state values inside the `AndroidView` factory block, causing rapid volume button presses to toggle between only two sizes. Using state resolution fixes continuous scrolling.

2. **Immediate Terminal Focus:**
   - The volume buttons were not registering until the screen was tapped because `TerminalView` did not have focus upon creation.
   - Added `terminalView.requestFocus()` inside the `AndroidView` factory to ensure hardware events route to the terminal immediately upon launch.

3. **Delete Connection Confirmation:**
   - Users reported accidentally swiping and deleting connections in `GroupedConnectionList.kt`.
   - Replaced immediate execution of `onDeleteConnection` with setting `profileToDelete` state. This triggers an `AlertDialog` confirming "Are you sure you want to delete '[nickname]'?" to prevent accidental data loss.

4. **Multi-method Authentication Fallback:**
   - Replaced the rigid `getAuthenticator` logic in `SshConnectionManager.kt` which previously selected a single method based on the profile's `AuthType`.
   - Created `CompositeAuthenticator` which iterates through a list of eligible authenticators. If an Identity Profile contains both a Private Key and a Password, it will attempt Key-based auth first, and upon failure or rejection by the server, silently fall back to Password-based auth.
   - Added `testHeadlessKeyConnection` to `SshConnectionManagerIntegrationTest` verifying key fallback behaviors.

## Verification Proof

- Volume key continuous font scaling verified locally on-device.
- Authentication fallback logic tested via unit test and manually verified.
