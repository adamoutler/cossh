# QA Proof for SSH-113

## 1. Implementation Details
The issue was that `SshConnectionManager` failed to parse manually entered PEM keys (specifically OpenSSH format) because the underlying `PEMParser` threw an exception when encountering proprietary OpenSSH blobs without proper byte-level extraction, and it also violated the strict **Volatile State Sanitization** invariant by converting the `ByteArray` to a `String`.

To fix this:
1. Created `PemUtils.parsePemToKeyPair(pemBytes: ByteArray)` which performs a pure byte-level linear scan to extract the base64 payload, stripping headers and footers without allocating any `String` objects into the JVM string pool.
2. The extracted base64 payload is decoded and then parsed into a `java.security.KeyPair`.
3. For `OPENSSH PRIVATE KEY`, we use `OpenSSHPrivateKeyUtil.parsePrivateKeyBlob` to extract the key parameters and convert them to `PrivateKeyInfo`. It securely extracts the public key parameters from the OpenSSH format as well.
4. `SshConnectionManager.loadKeyPairFromIdentity` was refactored to pass the `ByteArray` directly to `PemUtils`, completely removing the `String` conversion.

## 2. Test Validation
A new test `PemParsingTest.testParseOpenSshEd25519` was created with a valid OpenSSH key generated by `ssh-keygen -t ed25519`. The test verifies that `PemUtils` correctly parses the OpenSSH key into a valid `KeyPair` object with both public and private keys intact.

The test suite executed successfully.
