# Verification Proof for SSH-106

This ticket resolved navigation and connection lifecycle bugs across the Terminal screen.

**1. X Button Terminates Immediately:**
The `TerminalOverlayButtons`'s `onTerminate` handler was reverted to immediately trigger the `SshService.ACTION_DISCONNECT` intent without an extra UI dialog confirmation, closing the window rapidly as requested.

**2. Android Nav Back Gesture Asks:**
The `BackHandler` logic inside `TerminalScreen.kt` was simplified. By removing manual keyboard hiding logic that previously intercepted the gesture, `BackHandler` now consistently asks whether to Keep Alive or Terminate the SSH Session upon a swipe back.

**3. Terminate Physically Disconnects:**
A new `disconnect()` method was exposed in `SshConnectionManager`. The `SshService.stopSshConnection` method now explicitly calls `manager.disconnect()` to physically interrupt the PTY socket rather than simply cancelling the coroutine. This guarantees immediate termination.

**4. Notification Resumes Session:**
The `PendingIntent` inside `SshService.updateSessionNotification` was updated with `Intent.FLAG_ACTIVITY_SINGLE_TOP or Intent.FLAG_ACTIVITY_CLEAR_TOP`. This fixes the issue where tapping the notification failed to trigger the Intent parser in Compose's `AppNavigation`, ensuring it now successfully resumes active sessions instead of dropping the user onto the Connection List.
